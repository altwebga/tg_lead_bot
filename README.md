# Telegram Lead Bot

Telegram-бот для приема заявок на разработку, доработку сайтов и SEO/продвижение. Бот ведет пользователя по короткому сценарию: выбор услуги, описание задачи, бюджет и контакт. После заполнения заявка отправляется администратору в Telegram.

## Возможности

- Пошаговый сценарий заявки без внешнего хранилища
- Inline-кнопки для выбора услуги и бюджета
- Отправка заявки в админский чат
- Команды `/start` для новой заявки и `/cancel` для отмены
- Экранирование HTML в пользовательском вводе
- Повтор подключения к Telegram при временных сетевых ошибках
- Опциональное подключение через SOCKS-прокси

## Стек

- Node.js
- TypeScript
- grammy
- dotenv
- socks-proxy-agent для опционального SOCKS-прокси

## Установка

```bash
npm install
```

## Настройка

Создайте `.env` рядом с `package.json`:

```env
BOT_TOKEN=your_telegram_bot_token
ADMIN_CHAT_ID=your_admin_chat_id
```

`BOT_TOKEN` и `ADMIN_CHAT_ID` обязательны.

Если нужен прокси для подключения к Telegram, добавьте `PROXY_URL`:

```env
PROXY_URL=socks5://username:password@proxy_host:proxy_port
```

Поддерживаются схемы `socks4://`, `socks4a://`, `socks5://` и `socks5h://`. Если `PROXY_URL` не задан или оставлен пустым, бот подключается к Telegram напрямую.

## Структура проекта

```text
src/
  bot.ts        # Точка входа, сценарий заявки и запуск бота
  config.ts     # Проверка и экспорт переменных окружения
  keyboards.ts  # Inline-клавиатуры и callback-значения
  types.ts      # Типы контекста, сессии и заявки
```

## Сценарий заявки

1. Пользователь отправляет `/start`
2. Выбирает услугу
3. Описывает задачу
4. Выбирает бюджет
5. Оставляет контакт
6. Бот отправляет заявку администратору

## Прокси

Прокси не обязателен. Для прямого подключения просто не добавляйте `PROXY_URL` в `.env`.

Если логин или пароль прокси содержит спецсимволы, бот кодирует их перед созданием подключения. Например, пароль `pa:ss@word` можно указать прямо в URL, а конфиг приведет его к корректному виду для `socks-proxy-agent`.
